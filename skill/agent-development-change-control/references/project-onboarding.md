# Project Onboarding

Use this reference when a project does not yet have governance overlays.

## Create the first project governance set

Produce or refine:
- project brief
- protected core list
- contracts
- golden dataset expectations
- release checklist
- rollback baseline rule

## Questions to answer

- which files or functions preserve critical architecture behavior?
- which outputs or interfaces must remain stable?
- which user journeys are must-pass before release?
- what is the current stable baseline or rollback target?

## Output bias

Prefer a useful first draft over perfect completeness. Keep the first version concrete and reviewable.
